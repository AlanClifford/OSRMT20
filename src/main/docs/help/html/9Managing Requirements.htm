Managing Requirements

 

Once the features are defined, the detailed requirements need to be defined

The key attributes of this functionality are

-       Feature title

-       Detailed description of the requirement

-       Priority or criticality of the requirement

-       Dependency on any other requirement or functionality

-       Complexity ( medium, high, low)

-       Category ( Business requirement, data requirement, system requirement etc)

It is possible to define the background/ Context of this requirement.

It is also possible to define a use case for the requirement.

System allows to define any number of requirements.  It is possible to add//remove/update requirement at all stages of the software development. It is also possible to trace or track the changes made to a requirement during the life cycle of the project.
